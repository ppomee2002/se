# 1. 서론

## 1.1 목적 및 범위

본 문서는 클라우드 파일 공유 시스템인 **미니 드라이브**의 요구사항을 객체지향 분석 관점에서 정리하기 위한 요구사항 분석서이다.

미니 드라이브는 조직 내부 사용자가 파일을 중앙에서 저장, 관리, 검색, 공유할 수 있도록 지원하는 웹 기반 파일 관리 시스템이다. 기존에는 이메일, 메신저, 개인 클라우드 서비스 등을 통해 파일을 공유했기 때문에 최신 파일 관리가 어렵고, 파일 검색 시간이 오래 걸리며, 외부 협력 업체와 파일을 주고받을 때 보안 문제가 발생할 수 있었다.

본 시스템은 이러한 문제를 해결하기 위해 다음 기능을 제공하는 것을 목표로 한다.

- 사용자 계정 기반 로그인
- 파일 업로드 및 다운로드
- 폴더 기반 파일 관리
- 사용자 또는 그룹 단위 파일 공유
- 외부 공유 링크 생성
- 파일 검색
- 파일 버전 관리
- 삭제 파일 복구
- 관리자 계정 및 저장 공간 관리
- 권한 기반 접근 제어

본 문서에서는 위 기능을 중심으로 시스템의 주요 Actor, Use Case, 클래스 후보, CRC 카드, 동적 흐름, 인터페이스, 제약사항 및 요구사항 추적표를 정의한다.

## 1.2 용어 정의

| 용어 | 설명 |
|---|---|
| 미니 드라이브 | 조직 내부에서 사용하는 클라우드 기반 파일 저장 및 공유 시스템 |
| 사용자 | 시스템에 로그인하여 파일을 업로드, 다운로드, 검색, 공유하는 일반 직원 |
| 관리자 | 사용자 계정, 저장 공간, 시스템 정책 등을 관리하는 사람 |
| 외부 사용자 | 조직 외부에서 공유 링크를 통해 제한적으로 파일에 접근하는 협력 업체 또는 외부 인원 |
| 파일 | 문서, 이미지, PDF, 압축 파일, 영상 파일 등 시스템에 저장되는 데이터 단위 |

| 공유 링크 | 외부 사용자 또는 특정 사용자에게 파일을 제공하기 위해 생성되는 URL |

| 버전 관리 | 파일이 수정될 때 이전 상태를 저장하고 필요 시 복원할 수 있도록 관리하는 기능 |
| 접근 제어 | 사용자 권한에 따라 파일 접근 가능 여부를 판단하는 기능 |
| 삭제 파일 복구 | 삭제된 파일을 일정 기간 동안 보관하고 복원할 수 있도록 하는 기능 |

# 2. 시스템 개요

## 2.1 소프트웨어 컨텍스트

미니 드라이브는 사용자가 웹 브라우저를 통해 접속하는 클라우드 파일 관리 시스템이다. 사용자는 파일을 업로드하고 다운로드하며, 폴더를 생성하여 파일을 정리할 수 있다. 또한 다른 사용자 또는 외부 협력 업체와 파일을 공유할 수 있다.

관리자는 사용자 계정과 저장 공간을 관리하며, 시스템의 기본적인 운영 정책을 관리한다. 외부 사용자는 공유 링크를 통해 허용된 파일에만 제한적으로 접근한다.

---

### 2.1.1 Actor Table

| Actor | Role |
|---|---|
| 사용자 | 파일을 업로드, 다운로드, 검색, 공유하고 폴더를 관리하는 일반 직원 |
| 관리자 | 사용자 계정 생성/삭제, 사용자 저장 공간 관리, 시스템 운영 정책을 관리하는 사람 |
| 외부 사용자 | 공유 링크를 통해 제한된 파일을 다운로드하거나 확인하는 조직 외부 인원 |
| 시스템 | 인증, 파일 저장, 검색, 공유 링크 생성, 권한 확인, 버전 저장 등을 수행하는 소프트웨어 시스템 |
