# 1. 서론

## 1.1 문서의 목적 및 범위

이 문서는 조직 내부에서 사용할 수 있는 클라우드 파일 공유 시스템인 **미니 드라이브**의 요구사항을 정의하기 위한 문서이다.

미니 드라이브는 조직 내 여러 팀이 프로젝트 자료를 중앙에서 저장, 관리, 공유할 수 있도록 지원하는 웹 기반 파일 관리 시스템이다.

현재 조직에서는 문서, 설계 파일, 보고서, 이미지 등의 자료를 이메일, 메신저, 개인 클라우드 서비스 등을 통해 공유하고 있다. 이러한 방식은 파일의 최신 버전을 확인하기 어렵게 만들고, 필요한 파일을 찾는 데 많은 시간을 소요하게 하며, 외부 협력 업체와 파일을 공유할 때 보안 문제를 발생시킬 수 있다.

본 문서는 이러한 문제를 해결하기 위해 필요한 기능적 요구사항, 비기능적 요구사항, 인터페이스 요구사항, 개발 요구사항을 정의하는 것을 목적으로 한다.

## 1.2 대상 시스템 개요

### 1.2.1 프로젝트 정의

미니 드라이브는 조직 내부 사용자를 대상으로 하는 클라우드 기반 파일 저장 및 공유 시스템이다.

사용자는 웹 브라우저를 통해 파일을 업로드하고 다운로드할 수 있으며, 폴더를 생성하여 파일을 체계적으로 관리할 수 있다.

또한 사용자는 파일을 다른 사용자와 공유할 수 있고, 외부 협력 업체와도 공유 링크를 통해 파일을 전달할 수 있다.

시스템은 파일 검색, 권한 관리, 버전 관리, 사용자 계정 관리 기능을 제공하여 조직 내 협업 효율성과 파일 관리 안정성을 높이는 것을 목표로 한다.

### 1.2.2 주요 기능 설명

| ID | 주요 기능 | 설명 |
|---|---|---|
| F01 | 파일 업로드 | 사용자가 업무 파일을 시스템에 업로드한다. |
| F02 | 파일 다운로드 | 사용자가 저장된 파일을 다시 다운로드한다. |
| F03 | 폴더 관리 | 사용자가 폴더를 생성, 수정, 삭제하고 파일을 이동한다. |
| F04 | 파일 공유 | 사용자가 파일을 내부 사용자 또는 외부 사용자와 공유한다. |
| F05 | 권한 관리 | 파일 열람, 수정, 댓글 권한을 설정한다. |
| F06 | 파일 검색 | 파일 이름, 업로드 날짜, 사용자, 파일 유형 기준으로 검색한다. |
| F07 | 버전 관리 | 파일의 이전 버전을 조회, 다운로드, 복원한다. |
| F08 | 사용자 계정 관리 | 사용자가 로그인하고 관리자가 계정을 관리한다. |
| F09 | 보안 관리 | 인증, 권한 기반 접근 제어, 전송 암호화를 제공한다. |

## 1.3 용어 정의

| 용어 | 설명 |
|---|---|
| 미니 드라이브 | 조직 내부에서 파일을 저장, 공유, 검색, 관리하기 위한 클라우드 파일 공유 시스템 |
| 사용자 | 시스템에 로그인하여 파일을 업로드, 다운로드, 공유하는 조직 구성원 |
| 관리자 | 사용자 계정, 저장 공간, 시스템 권한 등을 관리하는 사람 |
| 외부 사용자 | 조직 외부에서 공유 링크를 통해 파일에 접근하는 협력 업체 또는 관계자 |
| 공유 링크 | 특정 파일을 외부 사용자에게 전달하기 위해 생성되는 접근용 URL |
| 권한 | 파일에 대해 열람, 수정, 댓글 작성 등을 수행할 수 있는 범위 |
| 버전 관리 | 파일이 수정될 때 이전 상태를 저장하고 필요 시 복원할 수 있도록 하는 기능 |
| 복구 | 삭제된 파일을 일정 기간 내에 다시 되돌리는 기능 |

## 4.2 요구사항 작성 기준

본 요구사항 정의서는 다음 기준을 따랐다.

- 요구사항은 기능적 요구사항, 비기능적 요구사항, 인터페이스 요구사항, 개발 요구사항으로 구분하였다.
- 각 요구사항은 “시스템은 ~해야 한다”, “사용자는 ~할 수 있어야 한다”와 같이 검토 가능한 문장으로 작성하였다.
- 각 요구사항에는 고유 식별자를 부여하였다.
- 각 요구사항에는 우선순위를 상, 중, 하로 구분하여 표시하였다.
- 고객 요구사항 문서에 제시된 파일 저장, 폴더 관리, 파일 공유, 파일 검색, 버전 관리, 사용자 계정, 보안, 성능, 향후 확장 기능을 기준으로 요구사항을 정의하였다.
