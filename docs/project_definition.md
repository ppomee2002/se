# 클라우드 파일 공유 시스템(미니 드라이브) 프로젝트 정의서

**프로젝트명**
사내용 파일 공유 시스템 구축 프로젝트

**개발 시스템명**
싱크스페이스 (SyncSpace)

## 프로젝트 배경
우리 조직은 여러 팀이 동시에 협업하는 프로젝트 기반 환경에서 문서, 설계 파일, 보고서, 이미지 등 다양한 자료를 이메일, 메신저, 개인 클라우드 서비스 등을 통해 공유하고 있다. 이러한 방식을 사용하면 최신 버전 관리의 어려움, 파일 탐색 시간 증가, 외부 공유 시 보안 문제를 발생시킬 수 있다. 따라서 조직 내부에서 사용할 수 있는 중앙 집중형 클라우드 파일 관리 및 협업 시스템이 필요하므로 클라우드 파일 공유 시스템을 자체적으로 구축한다.

## 핵심 기능
* **계정 및 보안 관리** 이메일/비밀번호 기반 로그인, 데이터 전송 암호화, 권한 기반 접근 제어 및 관리자의 사용자(계정/그룹/용량) 관리 기능
* **파일/폴더 관리** 문서, 이미지, PDF 등 일반 업무 파일 업/다운로드(메타데이터 포함) 및 폴더 생성, 이름 변경, 삭제, 이동 기능
* **파일 공유 및 권한 관리** 특정 사용자 대상 맞춤형 권한 부여(보기/수정/댓글) 및 만료 기간 설정이 가능한 외부 공유 링크 제공
* **파일 검색 기능** 파일 이름, 날짜, 업로더, 파일 유형 등 다중 조건 기준 검색 기능
* **버전 및 파일 관리** 이전 버전 목록 확인, 다운로드, 복원 기능 및 실수로 삭제한 파일 일정 기간 내 복구 기능

## 유사 소프트웨어 조사 및 분석

**Google Drive**
* 파일 및 폴더 업로드/공유 기능을 제공한다.
* 보기(Viewer), 댓글(Commenter), 편집(Editor) 권한을 구분할 수 있다.
* 파일 버전 관리 기능을 제공한다.
* 협업 문서 중심의 사용성이 강점이다.
* 권한 기반 공유 구조와 버전 관리 기능을 참고할 수 있다.

**OneDrive**
* 파일 공유 및 버전 히스토리 기능을 제공한다.
* 다양한 파일 형식에 대해 버전 복원이 가능하다.
* 조직/기업 계정 환경과의 연동에 유리하다.
* 조직 내부 계정 기반 관리 체계와 기업형 파일 관리 구조를 참고할 수 있다.
